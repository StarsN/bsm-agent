# 复盘提取数据的字段说明

`scripts/extract_review_data.py` 输出的 JSON 包含以下顶层字段：

## journal — 今日所有操作日志（开仓 + 平仓）

journal 的每行就是 journal 表的全部字段，按时间从早到晚排列。

  .id             — 自增主键
  .order_id       — 关联 trade_positions.id。同一仓位的开仓和平仓共享同一个 order_id，靠它配对
  .token          — 币种
  .action         — "open" 开仓 / "close" 平仓
  .price          — 操作价格：开仓价 or 平仓价
  .tier           — 仓位档位 full/half/quarter（仅开仓有，平仓 NULL）
  .stop_loss      — 止损价（仅开仓有）
  .tp1_price      — 止盈1（仅开仓有）
  .tp2_price      — 止盈2（仅开仓有）
  .reason         — 开仓理由 / 平仓理由
  .dimension_data — 市场快照 JSON。开仓时是 Agent 填的入场快照，平仓时是系统取的出场快照
  .market_overview— 市场环境一句话，如 "BTC +1.2%，恐惧贪婪65(贪婪)，欧亚重叠中等流动性时段"（仅开仓有）
  .lesson_checked — 开仓前查了哪些 lessons（仅开仓有）
  .pnl_pct        — 盈亏百分比（仅平仓有）
  .close_reason   — 平仓原因：sl_hit(止损) / tp_hit(跟踪止盈) / manual(取消收藏)（仅平仓有）
  .hold_duration  — 持仓时长，如 "2h30m" 或 "45m"（仅平仓有）
  .pending_decision_id — 关联 pending_decisions.id（仅开仓有）
  .source_round   — worker 第几轮（仅开仓有）
  .social_score   — 社交热度分（仅开仓有）
  .created_at     — 时间

## opens — 开仓记录

journal 中 action="open" 的子集，字段同上。

## closes — 平仓记录

journal 中 action="close" 的子集，字段同上。

## closed_positions — 今日平仓的持仓

对应 trade_positions 表。条件：status='CLOSED'、今天平仓、source='agent'。

  .id             — 仓位 ID（= journal.order_id）
  .token          — 币种
  .side           — "LONG"
  .status         — "CLOSED"
  .entry_price    — 入场价
  .exit_price     — 平仓价（其实就是 closed_at 时的 current_price，字段名有点误导）
  .pnl_pct        — 总盈亏百分比
  .realized_pnl   — 已实现盈亏 USD
  .margin_amount  — 保证金
  .leverage       — 杠杆倍数
  .quantity       — 持仓数量
  .stop_loss_price— 设定的止损价
  .tp1_price      — 设定的止盈1
  .tp2_price      — 设定的止盈2
  .signal_snapshot— 入场时的信号快照 JSON（含 Agent 的 dimension_data、risk_meta 等）
  .open_reason    — 开仓理由摘要
  .created_at     — 开仓时间
  .closed_at      — 平仓时间

## open_positions — 当前活跃持仓

status IN ('OPEN','PARTIAL') 的 Agent 持仓。字段与 closed_positions 类似，但没有 exit_price 和 closed_at。pnl_pct 是当前浮动盈亏。

## existing_lessons — 已有教训

  .id             — lessons 表 ID
  .token          — 币种，'*' 是全局教训
  .root_cause     — 根本原因
  .rule_update    — 衍生的规则（复盘时用来去重）
  .severity       — critical / warning / medium
  .created_at     — 什么时候写的

## existing_rules — 已有规则去重集合

existing_lessons 里所有 rule_update 去重后的列表。写新 lesson 前看这个，rule_update 内容一致就不重复写。

## tag_stats — 止损标签统计

系统自动给每笔止损打标签。这里是全量汇总：
  {"entry_funding_hot": 27, "buy_pressure_faded": 73, "oi15_reversed": 15, ...}

数字 = 历史上有多少笔止损命中该标签。高频标签 = 这个失败模式反复出现。
