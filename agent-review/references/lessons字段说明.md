# lessons 表字段

## 必填

| 字段 | 说明 | 写法 |
|------|------|------|
| token | 币种，或 `*` 表示全局 | `"FET"` / `"*"` |
| lesson | 具体教训 | 直接可操作，不是泛泛而谈 |
| signal_error | 信号层面哪里判断错了 | 具体的，不是"看错了" |
| what_missed | 遗漏了什么关键信号 | 具体的，不是"没注意" |
| root_cause | 根本原因一句话 | 追本溯源 |
| rule_update | 由此衍生的模式。三段式：情境+倾向+策略。不设数值硬阈值 | 见下方示例 |
| severity | critical / warning / medium | 按严重程度 |

## 亏损 vs 盈利的写法

亏损的 lessons 和盈利的 lessons 侧重点不同：

### 市场随机噪音

如果入场时所有市场数据均符合高胜率标准，仅因突发随机抛压导致止损，这是市场随机噪音，不是研判错误。写空条目，**禁止生成 `rule_update`**：

```json
{
  "token": "FET",
  "signal_error": "noise",
  "root_cause": "市场随机噪音",
  "lesson": "基本面与盘口未被破坏，属于正常概率损耗",
  "severity": "medium",
  "what_missed": "",
  "rule_update": ""
}
```

`rule_update` 留空、`what_missed` 留空。不为随机噪音修改系统规则。

### 亏损单（逻辑研判错误）

重点在"为什么错了"。signal_error 和 what_missed 是关键。

`rule_update` 用三段式结构：**情境 + 倾向 + 策略**，描述模式而非设死数字。

```
lesson:       "阻力位高位滞涨叠加负资金费率时，极易引发多头逼空，不应按常规阻力位逻辑左侧做空"
signal_error: "仅将'价格触及关键阻力位+K线滞涨'解读为确定性做空信号，错误估计了上方的抛压力量"
what_missed:  "遗漏了'资金费率呈微弱负值'与'盘口买方挂单密集'的共振，未察觉空头被算计的流动性陷阱"
root_cause:   "过度依赖单维度的图表形态（阻力位），忽视了衍生品特有的情绪与流动性反噬机制"
rule_update:
  情境: "价格在关键阻力位附近表现为高位滞涨，同时资金费率呈现负值（空头补贴），且盘口深度显示买方挂单密集"
  倾向: "这种'阻力位测试+空方筹码拥挤'的共振环境，极大概率不是见顶，而是洗盘后演变为暴力的向上轧空（Short Squeeze）突破"
  策略: "面对此类高危共振区，若出现顺势做多信号，还需判断其他信号，等价格和taker方向一致后再入场，或下调仓位档位"
severity:     "critical"
```

### 盈利单

重点在"什么判断对了，下次能不能复用"。

```
lesson:       "成功捕捉多空情绪极度分化背后的‘聪明钱’真实意图"
signal_error: ""
what_missed:  "" 或 "虽然判断对了方向，但在盈亏比计算上，未能充分考虑当时大盘 Beta 收益的加成作用，导致止盈位（TP2）设置偏保守"
root_cause:   "在散户和大户情绪出现背离时，没有被市场表象迷惑，而是坚定地将‘持仓量同步’作为最终确认信号"
rule_update:
  情境: "散户情绪指标（如广场热度或散户做多比例）极度偏空，但核心链上指标（如大户多空比或大单资金净流入）显示大户在坚定做多，同时该资产的多周期 OI（未平仓合约）呈现稳步增长态势"
  倾向: "当出现这种‘散户看空但大户真金白银买入且持仓量配合’的背离结构时，大户往往具备显著的信息优势或资金优势，后续发动单边上涨行情的胜率极高"
  策略: "在遇到此类共振信号时，优先考虑开仓，持仓过程关注大户方向是否转向"
severity:     "medium"
```

盈利单写 lesson 的价值在于：把"感觉这次不错"变成"这条模式经历史验证过"。未来开仓前查教训时，正面模式和负面模式都在视野里。

## 可选

| 字段 | 说明 |
|------|------|
| order_id | 关联 trade_positions.id |
| direction | long / short |
| entry_price / exit_price | 入场/出场价 |
| pnl_pct | 盈亏百分比 |
| market_snapshot | 入场时行情摘要 |
| macro_context | 市场环境（BTC走势、时段） |

## 质量标准

好的教训描述**模式和倾向**，差的教训要么太泛要么太死。

- ✅ "OI涨但价格横盘 + taker方向偏弱 → 警惕派发风险，入场需更多信号确认"
- ❌ "下次注意OI"（太泛）
- ❌ "OI 1h>5%且taker<1.0就不开"（太死：阈值在变化的市场里很快失效）

- ✅ 三段式 rule_update：情境 + 倾向 + 策略
- ❌ 一行硬规则："X>Y → 不开多"

## 为什么不鼓励硬阈值

市场条件会变。今天 20% 是追高，下周可能 10% 就是追高。硬阈值写过两周就得改，还会让 Agent 在边界上做出非黑即白的判断。
