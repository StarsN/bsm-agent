# Agent 决策报告模板

推送时用反引号代码块包裹，TG 会用等宽字体渲染，表格才能对齐。
不要发无代码块的裸文本。

```
🤖 Agent决策 {current_time} (Round {N})

候选池: {candidate_count} 有快照 / {total_candidates} 总候选 个
（candidate_count 和 total_candidates 来自提取脚本输出的 JSON 顶层字段）
市场: {一句话判断}

━━━━━━━━━━━━━━━━━━━━━━━━
持仓状态（{N}个）
┌────────┬────┬──────────┬──────────┬─────────┬──────────┬──────────┐
│ 币种   │ 方向│   入场   │   当前   │   PnL   │   止损   │   止盈   │
├────────┼────┼──────────┼──────────┼─────────┼──────────┼──────────┤
│ TOKEN  │LONG│ $x.xxxx  │ $x.xxxx  │ +x.xx%  │ $x.xxxx  │ $x.xxxx  │
└────────┴────┴──────────┴──────────┴─────────┴──────────┴──────────┘
账户: Equity $xxx | Available $xxx | Locked $xxx
━━━━━━━━━━━━━━━━━━━━━━━━

开仓:
  {TOKEN} {tier} @ ${entry} sl=${sl} tp1=${tp1} tp2=${tp2}
  — {理由}
  — 教训检查：{查了几条lessons，是否命中}

跳过:
  {TOKEN} — {原因}

⏸ 本轮无操作   （空决策时）
```

## 跳过原因分类

- `信号不足` — 判断不值得开
- `已有持仓` — 同币已有仓位
- `查教训命中` — 与历史失败场景吻合
- `无快照` — 无市场数据
- `流动性差` — 盘口太薄
