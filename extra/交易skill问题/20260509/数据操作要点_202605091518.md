## JSON 解析

`extract_market_data.py` 输出是单行压缩 JSON（46KB+），`read_file` 会截断。**禁止直接 Read**。

用 Python 脚本解析，写到 `/tmp/*.py` 再执行（`python3 -c` 会触发审批拦截）：

```python
import json
with open("/tmp/market_data.json") as f:
    d = json.load(f)
for c in d["candidates"]:
    print(f"{c['token']}: price={c['price']} ...")
```

## 市场数据字段

候选币的实时市场数据在**顶层短英文 key**。写 `dimension_data` 时把候选对象的全部字段 `JSON.stringify`，一个不漏，不要改字段名也不要删字段。系统入库时自动翻译为中文。

完整字段列表见 `snapshot字段说明.md`，共 30 个字段。

## 安全格式化

缺失数值在 JSON 中为 `null`（Python `None`）。格式化前判断 `is not None`：

```python
def sf(v, fmt=".2f"):
    """安全格式化：None 返回 '-'，正常数值按 fmt 格式化。"""
    if v is None:
        return "-"
    if isinstance(v, str):
        try:
            v = float(v)
        except ValueError:
            return v
    return f"{v:{fmt}}"
```

**不要**把 `None` 转成 `"?"` 或其他字符串再参与运算——会引发 `ValueError`。
⚠️ **格式说明符选用**：snapshot 字段（`change_1h_pct`、`oi_change_1h_pct`、`funding_rate_pct` 等）存储的值**已经是百分数形式**（如 8.07 表示 8.07%），用 `sf(v, '.2f')` 后手动加 `%`，**不要**用 `sf(v, '.2%')`（会再乘 100，导致 8.07 显示为 807%）。Taker 买卖比（`taker_buy_sell_ratio`）是无单位的比值，直接用 `.2f` 无需后缀。

```python
# ✅ 正确
print(f"1h={sf(c.get('1h'), '.2f')}%  taker={sf(c.get('taker'), '.2f')}  funding={sf(c.get('funding'), '.4f')}%")

# ❌ 错误 — .2% 把已是百分数的值再×100
print(f"1h={sf(c.get('1h'), '.2%')}")  # 8.07 → "807.40%"
```

## 持仓币种识别

`positions` 是 `trade_positions` 表的真实数据，`token` 字段为真实币名，不需要猜测。

## 决策 JSON 格式

`write_decisions.py` 要求信封格式，顶层必须包含 `market_read`：

```json
{"market_read": "BTC震荡偏多...", "decisions": [...]}
```

空决策也必须带 `market_read`，不能只写 `[]`。

## BTC 与 session 字段格式

`btc` 对象和 `session` 的字段名与候选币不同：

```python
# btc 用 chg_ 前缀
btc["chg_15m"]  btc["chg_1h"]  btc["chg_4h"]  btc["chg_24h"]
btc["funding"]  btc["oi_15m"]  btc["oi_1h"]  btc["oi_4h"]

# session 是纯字符串，不是 dict
session  # "美国时段（高流动性）" 直接使用，不要 .get('label')
```

⚠️ 不要用 `btc.get('1h')`（返回 None），BTC 用 `chg_1h`。不要对 `session` 调 `.get()`（会 AttributeError），直接当字符串用。

## 标签统计结构

`tag_stats` 是 `dict[str, int]`——标签名到出现次数的简单映射，**不是** `dict[str, dict]`：

```python
# ✅ 正确
for tag, count in tag_stats.items():
    print(f"  {tag}: {count}次")

# ❌ 错误（会导致 AttributeError）
tag_stats['buy_pressure_faded']['total']  # int 没有 .get()
```

如需详细的 win/loss 分布，用 `archive_lessons`。**注意类型可变**：`extract_market_data.py` 输出可能是 `list`（直接迭代）或 `dict[str, list]`（按 token 分组）。先 `isinstance` 判断再遍历：

```python
al = d.get("archive_lessons", [])
if isinstance(al, list):
    for entry in al:
        print(f"  token={entry.get('token')} tags={entry.get('tags')} result={entry.get('result')}")
elif isinstance(al, dict):
    for token, entries in al.items():
        for entry in entries:
            print(f"  {token}: tags={entry.get('tags')} result={entry.get('result')}")
```

## dimension_data 写入规则

`candidates` 和 `positions` 的 `token` 均为真实币名，没有被掩码。

`JSON.stringify(候选对象的全部字段)` 后，token 自然就是真实值，不要替换也不要删字段。

## `c["analysis"]` 可能全为 `None`时应该怎么做

⚠️ **字段位置**：`verdict`/`direction`/`tags`/`oi_divergence`/`notes`/`age` 在候选对象的**顶层**，不是 `c["analysis"]["verdict"]` 嵌套结构。直接用 `c["verdict"]`、`c["direction"]` 等访问。

当 worker 处于 `stage=scraping`（数据采集阶段）时，分析模块可能尚未运行，以上字段均可能为 `None`。但实际中也常已填充（worker 可能已完成前一轮分析），需逐字段判空。字段为 `None` 时必须完全依赖原始市场数据自行判断，不可因分析缺失而跳过分析环节。

## 第四步写决策 常见 reject 原因
**常见 reject 原因**：缺必填字段（action/token/tier/reason/source_round/social_score/mentions/dimension_data/market_overview/lesson_checked，缺一不可）、action 写成 `"side": "LONG"` 而非 `"open_long"`、dimension_data 未序列化为 JSON 字符串。被 reject 后必须补全所有字段重试。
