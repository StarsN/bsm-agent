SKILL 加载时系统自动加入的声明（你SKILL.md里没有的东西）

当系统加载一个 SKILL 时，`_build_skill_message()` (在 `agent/skill_commands.py` 第112行) 会在你的 SKILL.md 内容周围自动包裹以下几类声明，这些全是你写 SKILL.md 时没有的：

① 激活声明（activation note）
```
[IMPORTANT: The user has invoked the "xxx" skill, indicating they want
you to follow its instructions. The full skill content is loaded below.]
```
—— 每个 skill 被加载时开头都会自动加这一段，告诉你「这个 skill 已被激活」。

② Skill 目录注入
```
[Skill directory: /root/.hermes/skills/data-science/crypto-trading-signals]
Resolve any relative paths in this skill (e.g. `scripts/foo.js`,
`templates/config.yaml`) against that directory, then run them
with the terminal tool using the absolute path.
```
—— 系统自动提取 skill 的绝对路径，告诉 Agent 去哪里找关联文件。

③ Skill Config 注入（如果 SKILL.md frontmatter 声明了 `metadata.hermes.config`）
```
[Skill config (from /.hermes/config.yaml):
  api_timeout = 30
  retry_count = 3
]
```
—— 自动从 `config.yaml` 里读取对应的配置值注入进来。

④ 环境检查结果
```
[Skill setup note: Required environment setup was skipped...]
```
或
```
[Skill setup note: 环境变量 MY_API_KEY 未设置...]
```
—— 系统会扫描 SKILL.md frontmatter 里的 `required_environment_variables`/`required_commands`，自动检查是否满足，然后注入检查结论。

⑤ 支持文件列表
```
[This skill has supporting files:]
- references/api.md  ->  /root/.hermes/skills/xxx/references/api.md
- scripts/validate.py  ->  /root/.hermes/skills/xxx/scripts/validate.py

Load any of these with skill_view(name="xxx", file_path="<path>"),
or run scripts directly by absolute path (e.g. `node .../scripts/foo.js`).
```
—— 自动扫描 skill 目录下的 `references/`、`templates/`、`scripts/`、`assets/` 子目录，把实际文件路径列出来。

⑥ 模板变量展开（`${HERMES_SKILL_DIR}` / `${HERMES_SESSION_ID}`）
如果你 SKILL.md 里写了 `${HERMES_SKILL_DIR}`，系统会在加载时替换成真实绝对路径。如果你没写就没变化。

⑦ 内联 Shell 执行结果（`!`cmd``）
如果 `config.yaml` 里启用了 `skills.inline_shell: true`，SKILL.md 里 `!`date +%Y-%m-%d`` 这样的标记会被执行并替换为输出。默认为关。

---

## 还有一层：系统提示词里自动生成的 skill 索引

除了上面的 skill 加载包裹层，每次会话启动时，`build_skills_system_prompt()` 还会在系统提示词里自动生成一个：

```
## Skills (mandatory)
<available_skills>
  category-name:
    - skill-name: skill description from frontmatter
    - another-skill: its description
</available_skills>
```

这个列表完全自动生成，内容来自每个 SKILL.md 的 frontmatter 里的 `description` + `category` + `tags`。你的记忆区和当前聊天里看到的那个 skill 列表就是它。

---

总结逻辑链

```
skill_view(name="xxx") 
  → 读取 SKILL.md + frontmatter
  → preprocess_skill_content()  # 替换 ${VAR} 和执行 !`cmd`
  → 返回 JSON (含 content + skill_dir + setup_needed + linked_files 等元数据)

_build_skill_message(loaded_skill)
  → 包裹 activation note
  → 注入 skill directory 指引
  → 注入 skill config（如果有）
  → 注入 setup note（如果需要）
  → 注入 supporting files 列表
  → 最终拼成一段完整消息发给 LLM
```

你看到的那些 SKILL.md 里没有的「声明」，就是第②⑤步自动注入的。这是 Hermes 的设计哲学：SKILL.md 只写纯业务逻辑，环境上下文、路径映射、前置检查等基础设施层面的东西全部由系统自动处理。
