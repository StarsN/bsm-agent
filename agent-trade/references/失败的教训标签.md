# 失败的教训数据

提取脚本输出里有两个字段帮助你判断"这个场景历史上容易亏吗"。

## tag_stats — 失败标签统计

系统自动给每笔止损归档打标签。`tag_stats` 是所有历史止损单的标签汇总计数：

```json
{"entry_funding_hot": 27, "buy_pressure_faded": 73, "oi15_reversed": 15, ...}
```

数字 = 历史上有多少笔止损命中了这个标签。数字越大 = 这个失败模式出现越频繁。

| 常见标签 | 含义 |
|---------|------|
| `entry_funding_hot` | 入场时资金费率过高（多头拥挤） |
| `entry_lsr_hot` | 入场时散户多空比过高（情绪过热） |
| `entry_15m_hot` | 入场时 15m 涨幅过大（追涨） |
| `entry_1h_hot` | 入场时 1h 涨幅过大 |
| `buy_pressure_faded` | 平仓时买盘已经衰退（Taker 买盘消失） |
| `oi15_reversed` | 平仓时 15m OI 趋势反转 |
| `oi1h_reversed` | 平仓时 1h OI 趋势反转 |
| `oi4h_reversed` | 平仓时 4h OI 趋势反转 |
| `price_hit_stop` | 纯粹的价格止损（无其他标签） |
| `entry_not_healthy` | 入场时 verdict 不是"看起来健康" |

## archive_lessons — 最近止损记录

最近 10 条止损归档的详情：token、亏了多少(pnl)、失败原因(failed_reason)、标签(tags)。

用来快速扫一眼最近亏在哪些币上。

## 怎么用

开仓前看一眼 `tag_stats`——如果当前候选的特征和某个高频标签吻合（比如 funding 很高 → `entry_funding_hot` 命中 27 次），就得更谨慎。

这是系统自动打的标签，不是人工判断，可能有误。但高频标签通常不会骗人。
